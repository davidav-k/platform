# Параметры среды

Этот документ представляет собой официальные правила использования переменных окружения для локальной разработки на платформе. Перед запуском Docker Compose необходимо скопировать значения переменных из `.env.example` в `.env` . Ни в коем случае нельзя использовать значения переменных, предназначенные для производственной среды или общедоступной среды.

## Область применения

Инвентаризация включает в себя файлы формата Dockerfiles, файлы, связанные с технологиями Spring `application*.yml` и `bootstrap.yml` , репозиторий с настройками в формате native Config Server, расположенный по адресу `config/` , информацию об способах настройки параметров в Java, а также документацию по запуску репозитория.

`Required` означает, что соответствующая переменная должна присутствовать в корневом файле `.env` , согласно описанному пути запуска Docker Compose. Исключения могут предусматриваться в специальных примечаниях. Docker Compose передает значение `.env` нескольким контейнерам с использованием параметра `env_file` . Однако это не означает, что каждый контейнер будет использовать все значения этих переменных.

## Минимальные требования к местным стартапам

Создайте `.env` на основе `.env.example` , затем выполните следующую команду:

```bash
docker compose --env-file .env -f compose.yml up -d
```

Необходимые локальные значения:

*   `POSTGRES_HOST`, `POSTGRES_PORT`, `POSTGRES_DB`, `POSTGRES_USER`, `POSTGRES_PASSWORD`
*   `JWT_SECRET`, `ADMIN_PASSWORD`
*   `EMAIL_HOST`, `EMAIL_PORT`, `EMAIL_ID`, `EMAIL_PASSWORD`, `EMAIL_VERIFY_HOST`

Необязательные настройки на локальном уровне:

*   `APPLICATION_PORT`
*   `ACTIVE_PROFILE`
*   `CONFIG_SERVER_URI`
*   `EUREKA_URL`
*   `JWT_EXPIRATION`

Никогда не храните в производственной среде такие данные, как пароли, ключи для подписи JWT, учетные данные администраторов, учетные данные для работы с почтой, а также учетные данные для работы с сервисами, управляемыми сторонними организациями. `.env.example` представляет собой специальные символы, предназначенные исключительно для использования в процессе разработки.

### Редис

Redis запускается с помощью Docker Compose без необходимости настройки пользовательских переменных среды.

### Сервер настройки

| Имя | Необходимо. | По умолчанию | Пример | Поглощенный/Потребленный | Описание |
| --- | --- | --- | --- | --- | --- |

Клиенты сервера конфигурации по умолчанию используют URL сети Docker: `http://config-server:8888` . Значение `CONFIG_SERVER_URI=http://localhost:8888` необходимо указать при запуске пользовательского сервиса из среды разработки в отношении локального контейнера сервера конфигурации.

| Имя | Необходимо. | По умолчанию | Пример | Поглощенный/Потребленный | Описание |
| --- | --- | --- | --- | --- | --- |
| CONFIG\_SERVER\_URI | Нет. | http://config-server:8888 | http://localhost:8888 | Пользовательский сервис | Заменяет URL клиента конфигурационного сервера при запуске ИСП. |

### Эврика!

По умолчанию пользовательский сервис использует URL сети Docker Eureka: `http://eureka-server:8761/eureka` . Для запуска в среде IDE необходимо указать значение `EUREKA_URL=http://localhost:8761/eureka` .

| Имя | Необходимо. | По умолчанию | Пример | Поглощенный/Потребленный | Описание |
| --- | --- | --- | --- | --- | --- |
| EUREKA\_URL | Нет. | http://eureka-server:8761/eureka | http://localhost:8761/eureka | Пользовательский сервис | Заменяет URL-адрес сервиса Eureka, находящийся в ведении сервера конфигурации, при запуске интегрированных сред разработки. |

## Безопасность

### JWT

| Имя | Необходимо. | По умолчанию | Пример | Поглощенный/Потребленный | Описание |
| --- | --- | --- | --- | --- | --- |
| JWT\_SECRET | Да | Никто. | Ключ для подписи данных в формате Base64, предназначенный исключительно для целей разработки, от .env.example . | Службы обслуживания пользователей, выполнения задач, а также шлюз API. | Ключ для подписи JWT в формате Base64. | JWT\_EXPIRATION | Нет. | 432000 | 432000 | Пользовательский сервис | Необязательное значение времени истечения JWT, указанное в секундах. По умолчанию на сервере конфигурации это значение составляет пять дней. |
| ADMIN\_PASSWORD | Да | Никто. | dev\_example\_admin\_password\_change\_me | Пользовательский сервис | Пароль, используемый при создании локальной учетной записи администратора. |

### Куки/Файлы cookie

Имена куки задаются непосредственно в коде приложения. Атрибуты и сроки действия куки определяются по умолчанию, но также можно вносить изменения в эти настройки в зависимости от условий работы приложения.

| Имя | Необходимо. | По умолчанию | Описание |
| --- | --- | --- | --- |
| AUTH\_COOKIE\_SECURE | Нет. | false | Установите значение true для использования протокола HTTPS. |
| AUTH\_COOKIE\_SAME\_SITE | Нет. | Lax | Явное значение SameSite для куки автентификации. |
| AUTH\_ACCESS\_COOKIE\_MAX\_AGE\_SECONDS | Нет. | 600 | Время существования куки, связанной с доступом к сайту, в секундах. |
| AUTH\_REFRESH\_COOKIE\_MAX\_AGE\_SECONDS | Нет. | 7200 | Время существования куки, отвечающей за обновление данных в браузере, в секундах. |

## Внешние интеграции

### Kafka и переход уведомлений

| Имя | Необходимо. | По умолчанию | Пример | Поглощенный/Потребленный | Описание |
| --- | --- | --- | --- | --- | --- |
| KAFKA\_LOCAL\_PORT | Нет. | 9092 | 9092 | Docker Compose | Локальный порт Kafka broker на хосте. |
| KAFKA\_BOOTSTRAP\_SERVERS | Нет. | kafka:9092 | kafka:9092 | task-service, notification-service | Bootstrap servers для доступа сервисов к Kafka. |
| KAFKA\_TASK\_EVENTS\_TOPIC | Нет. | platform.task-events | platform.task-events | task-service, notification-service | Топик для доменных событий задач. |
| OUTBOX\_PUBLISHER\_ENABLED | Нет. | false | true | task-service | Включает polling outbox в task-service. По умолчанию выключено. |
| OUTBOX\_PUBLISHER\_ADAPTER | Нет. | logging | kafka | task-service | Выбирает адаптер публикации outbox. Значение kafka используется только при явном включении Kafka. |
| NOTIFICATION\_KAFKA\_ENABLED | Нет. | false | true | notification-service | Включает Kafka consumer в notification-service. По умолчанию выключено. |
| NOTIFICATION\_SERVICE\_ENABLED | Нет. | true | true | task-service | Включает вызовы notification-service из task-service. |
| NOTIFICATION\_SERVICE\_ASSIGNMENT\_REST\_ENABLED | Нет. | true | false | task-service | Управляет только синхронными REST-уведомлениями о назначении задач. Значение false используется при проверке перехода на Kafka. |
| NOTIFICATION\_SERVICE\_BASE\_URL | Нет. | http://notification-service:8087 | http://notification-service:8087 | task-service | Base URL для внутренних REST-вызовов notification-service. |
| NOTIFICATION\_SERVICE\_CONNECT\_TIMEOUT | Нет. | 2s | 2s | task-service | Таймаут подключения для внутренних REST-вызовов notification-service. |
| NOTIFICATION\_SERVICE\_READ\_TIMEOUT | Нет. | 2s | 2s | task-service | Таймаут чтения для внутренних REST-вызовов notification-service. |

Безопасная локальная проверка Kafka notification flow:

```text
OUTBOX_PUBLISHER_ENABLED=true
OUTBOX_PUBLISHER_ADAPTER=kafka
NOTIFICATION_KAFKA_ENABLED=true
NOTIFICATION_SERVICE_ASSIGNMENT_REST_ENABLED=false
```

По умолчанию `.env.example` оставляет Kafka consumer выключенным, а
синхронные REST-уведомления о назначении включенными.

### Будущие возможности интеграции

Для Prometheus, Grafana, сервисов по обработке электронной почты в производственных условиях, а также для интеграции с OpenAI не существует никаких действующих настроек, связанных с переменными окружения.

## Комментарии к настройкам резервных вариантов выполнения действий

Эти имена встречаются только в примерах использования функции Compose, сопровождаемых комментариями. Они не являются частью обязательных требований, предусмотренных спецификацией `.env` .

| Имя | Необходимо. | По умолчанию | Пример | Поглощенный/Потребленный | Описание |
| --- | --- | --- | --- | --- | --- |
| SPRING\_CLOUD\_CONFIG\_ENABLED | Нет. | Стандартные настройки для весеннего периода | false | Программы, предназначенные для работы весной, если соответствующие строки кода не отмечены как недействующие. | Предотвращает загрузку сервера настройок при запуске в режиме резервного варианта, когда используется только классопуть. |
| SPRING\_CONFIG\_IMPORT | Нет. | Настройка Bootstrap | optional:classpath:/application.yml | Программы, предназначенные для работы весной, если соответствующие строки кода не отмечены как недействующие. | Выбирается конфигурация классного пути для использования в случае неудачной попытки запуска программы. |

Сервер конфигурации является основным источником информации, необходимой для настройки работы сервисов на стадии их выполнения. Содержимое этого сервера включает лишь метаданные, необходимые для запуска сервисов, а также стандартные настройки, предусмотренные для удобства разработчиков. См. раздел «Управление конфигурацией».
