# Обслуживание пользователей

Внедрена служба для:

*   Регистрация пользователей
*   Аутентификация с использованием JWT
*   Управление ролями
*   Профили пользователей и жизненный цикл учетных записей
*   МФА
*   Управление сохранением данных в PostgreSQL с использованием механизмов, связанных с процессами передачи данных между узлами сети.

## Технологии

*   Java 17
*   Spring Boot 3.4.0
*   Spring Cloud 2024.0.1
*   PostgreSQL для хранения пользовательских данных
*   Маршрут для миграции схем
*   Google Guava для кэширования
*   Spring Security для обеспечения процедур аутентификации и авторизации
*   Spring Mail для получения уведомлений по электронной почте
*   Docker для контейнеризации
*   JUnit и Mockito для тестирования
*   PostgreSQL Testcontainers для тестирования процессов миграции и интеграции
*   Lombok: инструмент для сокращения количества кода, несущего лишнюю информацию.

## Конечные точки и разрешения

Это внутренние пути обслуживания. API-шлюз перенаправляет трафик пользователей с адреса `/api/users/**` на адрес `/api/v1/user/**` .

### Управление пользователями

*   POST `/api/v1/user/register` – Регистрация нового пользователя. Требуется: отсутствие необходимости в авторизации.
*   Получить `/api/v1/user/verify/account` – Проверка новой учетной записи пользователя с использованием специального ключа. Требуется: отсутствие необходимости в авторизации.
*   POST `/api/v1/user/login` – Подключение пользователя к системе. Требуется: отсутствие необходимости в авторизации.
*   POST `/api/v1/user/enable-mfa` – Включить функцию двухфакторной аутентификации для личного аккаунта авторизованного пользователя. Необходимо: действительный токен доступа.
*   POST `/api/v1/user/verify-mfa` – Проверка механизма двухфакторной аутентификации для пользователя. Требуется: отсутствие необходимости в дополнительной аутентификации.
*   POST `/api/v1/user/unlock` – Отпирание пользовательской учетной записи. Необходимо: `user:update`
*   POST `/api/v1/user/lock` – Блокировка учетной записи пользователя. Необходимо: `user:update`
*   POST `/api/v1/user/refresh` – Обновление данных доступа и токенов обновления. Необходимо наличие действительного токена обновления.
*   GET `/api/v1/user/profile` – Получение информации о профиле входящего в систему пользователя. Необходимо: действительный токен доступа.
*   PUT `/api/v1/user/{userId}` – Обновление данных пользователя. Необходимо иметь: `user:update` или быть владельцем учетной записи.
*   ПАЧКА `/api/v1/user/password/{userId}` – Изменение пароля пользователя. Необходимо иметь: `user:update` или быть владельцем учетной записи.
*   DELETE `/api/v1/user/{userId}` – Полное удаление пользователя и всех связанных с ним данных, находящихся в ведении служб, связанных с этим пользователем. Необходимо иметь права пользователя `user:delete` . Ограничения: удаление собственным пользователем или системным пользователем `0` запрещено.

API-шлюз обеспечивает возможность внешнего изменения паролей в формате `PATCH /api/users/password/{userId}` .

Подробную информацию о процедуре регистрации, входа в систему, обновлении данных, использовании токенов типа JWT и куки см. в разделе «Процесс аутентификации».

### Локальный запуск

Выполняйте команды из корневого каталога репозитория:

```bash
cp .env.example .env
docker compose --env-file .env -f compose.yml up -d --build
./scripts/check-local-stack.sh
```

Интерфейс MailHog доступен по адресу `http://localhost:8025` .

См. разделы, посвященные переменным окружения и стратегии миграции базы данных.

### Право собственности на конфигурацию

Сервер настроек является основным источником информации о настройках, необходимых во время выполнения программы. При запуске IntelliJ необходимо переопределить следующие переменные окружающей среды:

```text
CONFIG_SERVER_URI=http://localhost:8888
EUREKA_URL=http://localhost:8761/eureka
POSTGRES_HOST=localhost
EMAIL_HOST=localhost
```

Информацию о иерархии владельцев и режимах запуска см. в разделе «Управление конфигурацией».

### Создание и тестирование

```bash
mvn -B -f backend/user-service/pom.xml test
```

Тесты контроллеров и сервисов выполняются с использованием инструментов MockMvc и Mockito. При этом с помощью Testcontainers запускается PostgreSQL 16.1, выполняются действия, связанные с использованием инструмента Flyway, проверяются настройки, связанные с использованием фреймворка Hibernate, а также проверяется процесс удаления записей, принадлежащих сервису пользователя. Для выполнения этих тестов необходимо, чтобы Docker был запущен. Инструмент Maven Surefire определяет версию API Docker как `1.44` , чтобы обеспечить совместимость с Docker Engine 29.
